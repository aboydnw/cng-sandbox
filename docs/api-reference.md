# Ingestion API Reference

Read this when working on any endpoint under `/api/*`, adding new routes, debugging frontend ↔ backend contracts, or when the MCP server needs a new tool.

**Workspace header**: Workspace-listing reads (`GET /api/datasets`, `GET /api/stories`, `GET /api/connections`) require an `X-Workspace-Id` header identifying the caller's 8-character workspace. Without it the server returns 400. The Vite dev proxy and the MCP server both forward this header; direct API callers must set it themselves.

**Rate limiting**: Abuse-prone endpoints are rate-limited via slowapi (see `src/rate_limit.py`). Limits are keyed by `X-Workspace-Id` when present, otherwise by remote IP. Exceeding a limit returns 429 with a `Retry-After` header. A default ceiling of 300/minute applies globally; per-endpoint limits are noted alongside each route below. **Exception**: `POST /api/chat` is keyed by remote IP alone (never the client-supplied `X-Workspace-Id`), because that header is untrusted and rotating it would otherwise mint unlimited buckets for a paid endpoint.

**SSRF protection on URL fetches**: Endpoints that fetch user-supplied URLs (`/api/upload` via `convert-url`, `/api/connections`, `/api/inspect-url`, `/api/proxy`, `/api/zarr-proxy`, `/api/connect-remote`, `/api/discover`) call `validate_url_safe()` to block addresses in private, loopback, reserved, link-local, multicast, unspecified (`0.0.0.0`/`::`), and IPv4-mapped IPv6 ranges, and disable HTTP redirect following. Redirects are explicitly rejected (via `raise_if_redirect`) because the redirect target cannot be re-validated safely. For COG connections, an extra HEAD pre-flight runs before handing the URL to GDAL `/vsicurl/`, since GDAL follows redirects internally with no public knob to disable.

## Upload & conversion

- `POST /api/upload` — Upload a file (multipart form); returns 409 with `{"detail": "duplicate_dataset", "dataset_id": ..., "filename": ...}` if a file with the same name already exists in the workspace. Rate-limited to 20/hour. Accepted formats: GeoTIFF, GeoJSON, Shapefile `.zip`, NetCDF, HDF5, **LAS/LAZ point clouds**, and **GPX movement tracks**. LAS/LAZ files route through the point-cloud pipeline (`pointcloud_pipeline.py`): scan the header with laspy, convert to COPC with PDAL (`pdal translate … --writers.copc.forward=all`), validate (via the `las-to-copc` skill), store the `.copc.laz` to R2, and persist a `dataset_type: "pointcloud"` row with `copc_url` (a `/storage/…` path), `point_count`, WGS84 `bounds`, and `crs`. Point clouds are **not** registered in pgSTAC/tipg — the browser streams them directly. A LAS/LAZ file with no coordinate reference system hard-fails the job with a plain-language error ("This point cloud has no coordinate reference system. Re-export it with a CRS and try again."). GPX files route through the trajectory pipeline (`trajectory_pipeline.py`): MovingPandas parses the track points, writes a canonical GeoParquet plus a browser-ready `trips.json` sidecar (per-track `path`, epoch-ms `timestamps`, `speeds`), stores both to R2, and persists a `dataset_type: "trajectory"` row (`format_pair: "gpx-to-geoparquet"`) with `trips_url` (a `/storage/…` path), `track_count`, `point_count`, WGS84 `bounds`, and ISO `time_start`/`time_end`. Trajectories are **not** registered in pgSTAC/tipg — the browser fetches `trips.json` directly and animates it with a deck.gl `TripsLayer`. A GPX with no per-point timestamps, no track of ≥2 timestamped points, or over 2M points hard-fails with a plain-language error.
- `POST /api/convert-url` — Fetch and convert a file from a URL; same 409 duplicate response as above; rejects 3xx redirects from the source URL (returns 400). Rate-limited to 20/hour
- `GET /api/check-duplicate?filename=<name>` — Preflight duplicate check; returns 409 if a dataset with that filename exists, or `{"duplicate": false}` if not
- `POST /api/check-format` — Pre-upload format validation; accepts a file chunk and filename, returns `{"valid": true}` or `{"valid": false, "error": "..."}`
- `POST /api/upload-temporal` — Upload multiple raster files as a time series (2–50 files, same format). Rate-limited to 10/hour
- `POST /api/scan/{scan_id}/convert` — Trigger conversion after a scan completes

## Jobs

- `GET /api/jobs/{id}` — Get job status
- `GET /api/jobs/{id}/stream` — SSE stream of conversion progress; no workspace auth on this endpoint (EventSource cannot send custom headers); job UUIDs are the only access barrier — a scoped auth token or cookie-based workspace auth would be more robust for production

## Datasets

- `GET /api/datasets` — List datasets belonging to the caller's workspace. Requires `X-Workspace-Id` header (returns 400 without it). Master `is_example=True` rows are **not** returned directly; each workspace instead owns editable clones flagged `is_example_copy=True` (seeded via `POST /api/workspaces/{id}/examples`, see [Workspace examples](#workspace-examples)). Each dataset includes `expires_at` (ISO timestamp): the stored expiry if set, otherwise `created_at` + the 30-day retention window; always `null` for example copies, which never expire.
- `GET /api/datasets/{id}` — Get dataset metadata (includes `tile_url`, `is_example`, `is_shared`, and `expires_at`); returns 404 if the caller's workspace does not own the dataset and the dataset is not an example, not explicitly shared, and not referenced by a published story
- `PATCH /api/datasets/{id}` — Update editable dataset metadata (currently just `title`, 1–200 chars; pass `null` to clear); returns 403 if the dataset is an example or belongs to another workspace
- `PATCH /api/datasets/{id}/share` — Toggle public sharing; body `{"is_shared": true|false}`; returns 403 if the dataset is an example or belongs to another workspace
- `DELETE /api/datasets/{id}` — Delete a dataset; returns 403 if the dataset is a master example (`is_example=True`) or belongs to another workspace. A per-workspace example clone (`is_example_copy=True`) is deletable — it is removed with a shallow DB-row delete (no pgSTAC/R2 teardown, since clones reuse the master's shared storage)
- `PATCH /api/datasets/{id}/categories` — Update category labels and/or colors for a categorical raster; body is a list of `{"value": int, "label"?: str, "color"?: "#RRGGBB"}` objects (each entry must include at least one of `label` or `color`); the first color override snapshots the prior color into `defaultColor` so the UI can offer a reset; returns 400 if dataset is not categorical or a value doesn't exist, 403 if the dataset is an example
- `POST /api/datasets/{id}/mark-categorical` — Promote a non-categorical integer raster to categorical by scanning unique values and assigning default colors from the qualitative palette; returns 400 if values can't be extracted (unsupported dtype or too many unique values), 403 if the dataset is an example or belongs to another workspace
- `POST /api/datasets/{id}/unmark-categorical` — Demote a categorical raster back to continuous by clearing `is_categorical` and `categories` from metadata; returns 409 if the dataset is not currently categorical, 400 if the dataset is not a raster, 403 if the dataset is an example or belongs to another workspace
- `PATCH /api/datasets/{id}/render-mode` — Persist the user's chosen render mode (`"client"`, `"server"`, or `null` to clear); body `{"render_mode": "client"|"server"|null}`; returns 400 if `"client"` is requested but eligibility checks fail (e.g. no COG URL, temporal dataset, oversized file), 403 if the dataset is an example or belongs to another workspace
- `PATCH /api/datasets/{id}/colormap` — Persist the dataset's preferred colormap; body `{"preferred_colormap": "terrain"|"viridis"|...|null, "preferred_colormap_reversed": true|false|null}`; setting `preferred_colormap` to `null` also clears `preferred_colormap_reversed` regardless of payload; colormap name is case-insensitive and validated against the backend allowlist (returns 422 for unknown names); returns 400 if the dataset is not a raster, 403 if the dataset is an example or belongs to another workspace
- `GET /api/datasets/{id}/timeseries?lon=<lon>&lat=<lat>` — Return per-timestep pixel values for a temporal raster dataset at a given point; proxies to titiler-pgstac `/collections/{collection_id}/point/{lon},{lat}` for each timestep and caches complete (no-null) results in an in-process LRU cache; returns `[{datetime, value}]`; returns 400 if the dataset is not temporal or has no timesteps, 404 if the dataset doesn't exist or isn't accessible to the caller's workspace
- `GET /api/datasets/{id}/histogram?bins=<n>` — Return histogram data for a raster dataset; `bins` defaults to 20, range 2–100; for continuous rasters returns `[{bin_min, bin_max, count}]`; for categorical rasters returns `[{class, label, count}]` sorted by class value; proxies to titiler-pgstac `/collections/{collection_id}/statistics`; returns 502 if titiler fails, 404 if the dataset doesn't exist or isn't accessible

## Stories (shareable map narratives)

- `POST /api/stories` — Create a story with chapters linking to datasets
  - Chapters are a discriminated union on `type`. A `type: "flyover"` chapter (scroll-scrubbed 3D camera flight) carries: `keyframes` (array of `{ center: [lng, lat], zoom, bearing, pitch, caption? }`, ≥2 to render as a flight; fewer degrades to a plain map chapter), `map_state` (basemap + 3D flags; pose fields are ignored — keyframe 0 is the entry camera), optional `layer_config` (omit for the common terrain/buildings scene-setter — a bound data layer disables terrain), and `scroll_length` (number, default `1`; scroll-distance multiplier, ~one viewport height per keyframe segment). Same shape accepted by `PATCH /api/stories/{id}`.
- `GET /api/stories` — List stories belonging to the caller's workspace. Requires `X-Workspace-Id` header (returns 400 without it). Master `is_example=True` rows are **not** returned directly; each workspace instead owns editable clones flagged `is_example_copy=True` (seeded via `POST /api/workspaces/{id}/examples`, see [Workspace examples](#workspace-examples)). Each story includes `expires_at` (ISO timestamp, `created_at` + the 30-day retention window); always `null` for example copies, which never expire.
- `GET /api/stories/examples` — Public list of example stories (`is_example=True`); no `X-Workspace-Id` header required. Used by the public landing page to render the example carousel without provisioning a workspace.
- `GET /api/stories/{id}` — Get a story by ID
- `PATCH /api/stories/{id}` — Update a story; returns 403 if the story is an example (`is_example=True`)
- `POST /api/stories/{id}/fork` — Fork a story (clones chapters/metadata into a new story owned by the caller's workspace with `published=False` and `is_example=False`)
- `GET /api/stories/{id}/export/config` — Return the story as a portable `cng-rc.json` config (Pydantic-validated `CngRcConfig`). Resolves layer references to durable URLs: connection-backed layers expose `source_url=connection.url` with `cng_url=null`; dataset-backed layers expose `source_url=dataset.source_url` (often `null`) and `cng_url=dataset.cog_url || dataset.parquet_url`. Access mirrors `GET /api/stories/{id}` plus a third allowed branch: example stories (`is_example=True`) are accessible to every workspace even when unpublished. Returns 404 if the story does not exist or the caller is not entitled to view it.
- `POST /api/stories/{id}/export/interactive` — Build a self-contained interactive HTML bundle of the story and return it as a `.zip` (`application/zip`, `Content-Disposition: attachment`). Accepts an optional `scrolly_pngs` multipart field — a list of pre-rendered scrolly map screenshots whose filenames (minus extension) match the chapter IDs that the server cannot render headlessly. The endpoint consumes the same portable `CngRcConfig` produced by `/export/config` (via `story_export.build_config`) so manifests carry `opacity`, `colormap_reversed`, `rescale_min/max`, `band`, and `timestep` per layer. Image/video chapter media are pulled into `assets/<chapter>-image.<ext>` via `asset_inline.fetch_into` (reuses the same SSRF rules as `/api/connections`); YouTube thumbnails are derived from `video_id`. Chart chapters are pre-resolved server-side: `dataset_timeseries` reuses `cached_timeseries` from `routes/dataset_charts.py`, `dataset_histogram` calls titiler `/statistics`, and `csv_asset` reads bytes from R2 (mirroring `story_assets.py`). The server orchestrates per-chapter export via `src/services/interactive_export/` (raster pyramid tiling, vector → Arrow conversion, chart CSV, scrolly PNG packaging) and assembles the static HTML shell. Access rules mirror `GET /api/stories/{id}/export/config`. Returns 400 for unsupported inputs (e.g. zarr layers, flyover chapters, missing scrolly snapshots, oversized assets, layer types not yet supported in interactive export, unresolvable chart sources, unreachable raster/vector layer sources surfaced as `raster source unavailable` / `vector source unavailable` instead of an opaque 500) and 404 if the story is not visible to the caller.
- `DELETE /api/stories/{id}` — Delete a story; returns 403 if the story is an example

## Story assets (image/CSV uploads attached to stories)

Binary assets uploaded to support story chapters (image chapters and chart chapters). Images are compressed via Pillow on upload — the original is downscaled to a max long-edge of 2400px and a thumbnail is generated at 400px long-edge. Both are stored in R2 under `story-assets/{workspace_id}/{asset_id}/` and the metadata row lives in the `story_assets` table. CSVs are stored as-is at `story-assets/{workspace_id}/{asset_id}/data.csv`.

- `POST /api/story-assets` — Upload an asset (multipart form: `file`, `kind="image"|"csv"`, optional `story_id`). **Image** (`kind="image"`): must be `image/jpeg`, `image/png`, or `image/webp`, max 25 MB; returns `{asset_id, url, thumbnail_url, width, height, mime, size_bytes}` with status 201; returns 413 if over 25 MB, 415 if MIME not allowed, 400 if Pillow can't decode it. **CSV** (`kind="csv"`): max 5 MB, parsed with pandas; returns `{asset_id, url, thumbnail_url: null, mime, size_bytes, columns, row_count}` with status 201; returns 413 if over 5 MB, 415 if not a CSV MIME type, 400 if pandas can't parse it or the file has no columns. Requires `X-Workspace-Id` header.
- `GET /api/story-assets/{asset_id}` — Fetch asset metadata (`asset_id`, `kind`, `url`, `thumbnail_url`, `width`, `height`, `mime`, `size_bytes`, `row_count`, `columns`). `X-Workspace-Id` header is optional: public/unscoped assets are accessible without it; for workspace-scoped assets, the header must match the asset's workspace or the response is 404.
- `GET /api/story-assets/{asset_id}/data` — Stream the raw bytes of the asset's primary file (CSV body or original image), using the asset's stored MIME type as `Content-Type`. Lets the browser fetch asset bytes through the ingestion API instead of hitting R2 directly, which avoids needing CORS configured on the bucket. `X-Workspace-Id` header is optional with the same scoping rules as the metadata endpoint.
- `DELETE /api/story-assets/{asset_id}` — Delete an asset and its R2 objects (original + thumbnail). Returns 204 on success, 404 if the asset belongs to another workspace. Requires `X-Workspace-Id` header.

## Connections (external tile sources)

- `GET /api/connections` — List connections in the workspace. Requires `X-Workspace-Id` header (returns 400 without it). Master `is_example=True` rows are **not** returned directly; each workspace instead owns editable clones flagged `is_example_copy=True` (seeded via `POST /api/workspaces/{id}/examples`, see [Workspace examples](#workspace-examples)).
- `POST /api/connections` — Register an external data source (XYZ raster/vector, COG, PMTiles, GeoParquet, Zarr); validates the URL with `validate_url_safe` (blocks private/loopback IPs) and rejects HEAD redirects on the size probe. COG connections automatically run categorical detection and persist `is_categorical` + `categories` on the connection row; a HEAD request is also issued to capture `Content-Length` and store it as `file_size`, which gates client-side render eligibility. GeoParquet connections support two render paths via the optional `render_path` field: `"client"` (DuckDB-WASM) or `"server"` (tippecanoe → PMTiles → R2, async background job). When omitted, the server infers the path by issuing a HEAD request for the file size: files over 50 MB → `"server"`, otherwise → `"client"`. Zarr connections accept an optional `config` JSON object (persisted as-is on the connection row) used by the frontend renderer to remember the chosen variable, time dimension, and rescale range. Zarr connections also accept an optional `geozarr_attrs` object (the four single-resolution required GeoZarr keys: `spatial:dimensions`, `spatial:transform`, `spatial:shape`, `proj:code`); when present, the server validates its shape (returns 422 on bad input) and the frontend renderer passes it through as the deck.gl-zarr `metadata` prop so stores that lack inline GeoZarr metadata can still render. COPC connections (`connection_type: "copc"`) point at a remote `.copc.laz` point cloud that the browser streams directly (via `maplibre-gl-lidar` + `copc.js`) — there is no tile server. They persist an optional `config` (`color_mode` ∈ `elevation`/`intensity`/`classification`/`rgb`, `point_size`) and `bounds`; `PATCH /render-mode` to `client` is rejected because point clouds are not client-rendered COGs, and they are excluded from interactive HTML export (same 400 as zarr). Rate-limited to 30/hour
- `GET /api/connections/{id}/stream` — SSE stream of server-side conversion progress for a GeoParquet connection; emits `event: status` events with `{status, tile_url, error, feature_count}`; no workspace auth on this endpoint (EventSource cannot send custom headers); connection UUIDs are the only access barrier — a scoped auth token or cookie-based workspace auth would be more robust for production
- `GET /api/connections/{id}` — Get a connection by ID; returns 404 if the caller's workspace does not own the connection and it is not explicitly shared or referenced by a published story
- `PATCH /api/connections/{id}/share` — Toggle public sharing; body `{"is_shared": true|false}`; returns 403 if the connection belongs to another workspace
- `PATCH /api/connections/{id}/categories` — Update category labels and/or colors for a categorical COG connection; body is a list of `{"value": int, "label"?: str, "color"?: "#RRGGBB"}` objects (each entry must include at least one of `label` or `color`); the first color override snapshots the prior color into `defaultColor`; returns 400 if connection is not categorical or a value doesn't exist
- `PATCH /api/connections/{id}/render-mode` — Persist the user's chosen render mode (`"client"`, `"server"`, or `null` to clear); body `{"render_mode": "client"|"server"|null}`; returns 400 if `"client"` is requested but eligibility checks fail, 403 if the connection belongs to another workspace
- `PATCH /api/connections/{id}/colormap` — Persist the connection's preferred colormap; body `{"preferred_colormap": "terrain"|"viridis"|...|null, "preferred_colormap_reversed": true|false|null}`; setting `preferred_colormap` to `null` also clears `preferred_colormap_reversed` regardless of payload; colormap name is case-insensitive and validated against the backend allowlist (returns 422 for unknown names); returns 400 if the connection is not raster (only `cog`, `xyz_raster`, and raster `pmtiles` qualify), 403 if the connection belongs to another workspace
- `PATCH /api/connections/{id}/geozarr-attrs` — Set or clear the GeoZarr metadata override for a zarr connection; body `{"geozarr_attrs": {"spatial:dimensions": [..], "spatial:transform": [..], "spatial:shape": [..], "proj:code": "EPSG:..."} | null}`; passing `null` clears the override. Validates the shape of the four required keys (returns 422 on bad input); returns 400 if the connection is not a zarr connection, 403 if the connection is an example or belongs to another workspace
- `DELETE /api/connections/{id}` — Delete a connection

## Workspace examples

Per-workspace copies of the master `is_example=True` set (datasets, connections, stories). Cloning is metadata-only — clones reuse the master's `tile_url`/`stac_collection_id`/`source_url`, so no pgSTAC/R2 work happens on seed or removal. Seed/removed state lives in the `workspace_example_state` table so auto-seed fires once per workspace and a workspace that removed examples is not re-seeded. See [docs/example-data.md](docs/example-data.md). `{workspace_id}` must pass `validate_workspace_id` (8-character workspace).

- `GET /api/workspaces/{workspace_id}/examples` — Report seeding state; returns `{"state": "seeded"|"removed"|null}` (`null` when the workspace has no state row yet).
- `POST /api/workspaces/{workspace_id}/examples` — Clean-slate clone every master into the workspace (datasets → connections → stories, rewriting story chapter `dataset_id`/`connection_id` references to the cloned ids); also used to "add back" after removal. Returns `{"state": "seeded", "story_id_map": {...}}` mapping master story ids to the cloned ids.
- `DELETE /api/workspaces/{workspace_id}/examples` — Delete the workspace's example clones (shallow DB-row deletes). Returns `{"state": "removed", "deleted": {...}}`.

## Remote data discovery

- `POST /api/discover` — Discover geospatial files at a URL or S3 prefix. Rate-limited to 30/hour
- `POST /api/connect-remote` — Connect remote files as a mosaic or temporal dataset. Rate-limited to 30/hour
- `POST /api/connect-source-coop` — Register a curated source.coop product as a zero-copy pgSTAC collection (v1 products: `ausantarctic/ghrsst-mur-v2`, `alexgleith/gebco-2024`, `vizzuality/lg-land-carbon-data`, `vida/google-microsoft-osm-open-buildings`). Raster products (`kind="mosaic"`) run a STAC/path enumerator and register as pgSTAC mosaics; PMTiles products (`kind="pmtiles"`) read the remote PMTiles v3 header for bounds/zoom and register as a `FormatPair.PMTILES` vector dataset pointing at the source URL.

## Reader chat agent ("Ask this map")

Reader-facing conversational agent scoped to a single published or example story. The backend is a thin relay: it assembles a deterministic, cacheable per-story system prompt (built from the same reads as the story exporter, in `src/services/chat_context.py`) plus 7 tool definitions, then proxies the Anthropic Messages API stream (`src/services/chat_relay.py`). It never executes a tool — the browser runs the client-side tools (`fly_to`, `go_to_chapter`, `set_layer_visibility`, `highlight_location`, `query_point`, `get_area_statistics`, `query_features`) and drives the map. Disabled by default; active only when `CHAT_ENABLED=true` and `ANTHROPIC_API_KEY_CHAT` is set (chat settings live in `src/config.py`: `chat_model`, `chat_max_tokens`, `chat_max_turns`, `chat_daily_token_budget`, `chat_rate_limit`).

- `GET /api/chat/config` — Probe returning `{"enabled": bool}` so the frontend can show or hide the chat entry point. No auth.
- `POST /api/chat` — SSE relay; body `{"story_id": str, "messages": [...]}`. Streams `text`, `tool_use`, `done`, and `error` events (the browser executes any `tool_use` locally). Returns 404 when chat is disabled or the story is not published/example, 400 if the conversation exceeds `chat_max_turns`, and 503 when the in-process daily output-token budget (`chat_daily_token_budget`, resets on UTC date rollover) is exhausted. Rate-limited to 10/minute **keyed by remote IP alone** (not `X-Workspace-Id`; see the rate-limiting note above).

## Other

- `POST /api/bug-report` — Submit a bug report (creates a GitHub issue). Rate-limited to 5/hour
- `POST /api/inspect-url` — Inspect a remote URL before registering it as a connection; body `{"url": "..."}`; returns `{format, is_cog, size_bytes, bounds, crs, has_errors, error_detail}`. Format is detected from the path/template (`xyz`, `pmtiles`, `parquet`, `copc`, `cog`, `tiff`, `geojson`, or `unknown`; `.copc.laz`/`.laz` → `copc`). For non-XYZ URLs, runs SSRF validation (`validate_url_safe`) and a HEAD probe for `content-length` with redirects disabled (3xx responses are surfaced via `error_detail`). When the detected format is `tiff` and the HEAD probe succeeds, also runs `check_remote_is_cog` (10s timeout) to refine the `is_cog` flag; probe failures are swallowed and surfaced as `is_cog=false` rather than as errors. For `copc`, issues a ranged GET for the LAS header region (VLRs, not the trailing EVLRs) and returns the reprojected WGS84 `bounds` and `crs` (falling back to `null` on a header/CRS read failure); `bounds`/`crs` are `null` for all other formats. Rate-limited to 120/hour
- `GET /api/proxy` — Proxy GET requests to external URLs (used by the frontend for CORS-restricted resources); HTTPS-only, blocks private/loopback IPs, restricts to `.pmtiles`/`.tif`/`.tiff` extensions, rejects redirects, caps responses at 50 MB. Rate-limited to 120/hour
- `GET /api/zarr-proxy` — Pass-through proxy for objects in a public zarr v2/v3 store (used by the frontend when a store's bucket lacks CORS); HTTPS-only, blocks private/loopback IPs, forwards the client's `Range` header, returns 206 + `Content-Range` for partial reads, rejects upstream 3xx redirects, caps responses at 50 MB. No file-extension allowlist (zarr keys are arbitrary — chunks like `0.0.0`, metadata like `zarr.json` / `.zarray` / `.zmetadata`). Rate-limited to 600/hour to accommodate chunk fan-out at render time.
- `GET /api/health` — Health check
